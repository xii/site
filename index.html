xii definition example:

ceph:
  type: network
  mode: nat
  ipv4:
    ip: 192.168.111.1
    dhcp:
      start: 192.168.111.2
      end: 192.168.111.254

nodes:
  type: node
  count: 5
  network: ceph
  image: {{ image }}
  ssh:
    copy-key:
      users: [ceph, root]
  user:
    ceph: { password: ceph }
    root: { password: linux }

keys-on-all-nodes:
  type: distribute
  ssh-keys:
    exchange-all:
      hosts: nodes
      users: [root, ceph]

provisioning:
  type: ansible
  inventory:
    basic: nodes
    mon: nodes-1
    rgw: nodes-2
  run: deploy-ceph.yml

XII IS A VIRTUAL ENVIRONMENT MANAGER

Create virtual machines with there own network settings and let them provision with just one command is xii's goal.

With libvirt and libguest xii is build on a strong and reliable foundation to provide a commandline tool to rule virtual environments

It's just as easy as you can think to start:

$ xii create

Or create a more than one node and add a additional network

$ xii create --nodes nodes:3 --network ceph:nat

You can always edit the created file as you like!

Start you newly described environment with

$ xii -dimage=/path/to/image.qcow2 start

Easy right? And yes you can also hardcode the image variable or load it from environment
